# Memory Skill

**Why plaintext over RAG/vector memory:**
Vector embeddings are opaque — you cannot read, audit, or diff them. Markdown memory is human-readable: you can inspect what the agent "knows," detect poisoned or incorrect records, and edit directly. This is not just a UX preference — it is a security property. You cannot defend memory you cannot inspect.

**Rule #1: Verbatim over summary.**
Preserve exact names, paths, and decisions. Do not paraphrase or compress important facts.

**Rule #2: Structure over flat.**
Organized sections are retrieved better than a wall of text — both by humans and by AI.

**Rule #3: Update, don't replace.**
If CONTEXT.md exists, read it first and merge new findings. Do not overwrite existing valid entries.

---

## Step 1 — Check if CONTEXT.md exists

```bash
ls CONTEXT.md 2>/dev/null && echo EXISTS || echo MISSING
```

- **EXISTS** → go to Step 2 (update mode)
- **MISSING** → go to Step 3 (create mode)

---

## Step 2 — Update mode (CONTEXT.md exists)

Read the existing file:

```
Read CONTEXT.md
```

Then gather what's changed since it was written:

```bash
# What changed recently
git log --oneline -20
git diff HEAD~5 HEAD --stat 2>/dev/null | head -30
```

Read any files listed in recent commits that are relevant to the sections below.

Merge new findings into the existing CONTEXT.md — add, update, or remove entries. Do not delete entries without a reason. Preserve the structure.

Write the updated file and go to Step 5.

---

## Step 3 — Create mode (CONTEXT.md missing)

Gather project context. Run these in order:

### 3A — Read project manifest files

```bash
# Detect project type
ls package.json pyproject.toml go.mod Cargo.toml composer.json Gemfile 2>/dev/null
```

Read whichever manifest exists. Note: language, framework, main dependencies.

### 3B — Read CLAUDE.md if present

```bash
ls CLAUDE.md 2>/dev/null && echo EXISTS || echo MISSING
```

If it exists, read it — it contains project-specific rules and structure.

### 3C — Read recent git history

```bash
git log --oneline -30
git diff HEAD~10 HEAD --stat 2>/dev/null | head -40
```

Note: what has been actively worked on, what features were recently added or changed.

### 3D — Scan top-level structure

```bash
ls -1
```

Note the main directories and their purpose.

### 3E — Read the entry point(s)

Read the main entry file (e.g. `main.py`, `index.ts`, `app.py`, `cmd/server/main.go`). Do not read more than 3 files unless structure is unclear.

---

## Step 4 — Write CONTEXT.md

Write the file using exactly this structure. Fill every section. If a section has nothing, write "None identified."

```markdown
# [Project Name] — Context

> Generated by /memory on [YYYY-MM-DD]. Update by running /memory again.

## Architecture

[2–4 sentences: what the project does, its core components, and how they connect.
Be specific: name the framework, the DB, the key services. No generic descriptions.]

## Key Decisions

[Bullet list of non-obvious choices and why they were made. Examples:
- Chose X over Y because Z (performance, simplicity, team constraint)
- Auth uses JWT, not sessions — stateless for horizontal scaling
- No ORM — raw SQL for performance-critical queries
Only include decisions that are not obvious from the code.]

## Known Limitations & Tech Debt

[What's incomplete, what corners were cut, what needs attention.
Be specific: "User search does not support partial matches" not "search is limited".]

## Active Work

[What is currently being built or changed, based on recent git history.
Format: - [feature/area]: [what's happening]]

## Do Not Touch

[Files or areas that are fragile, owned by someone else, or off-limits without extra care.
If nothing: "None identified."]

## File Map

[Short map of key directories and what lives in them.
Only include non-obvious ones — skip standard framework dirs like node_modules, __pycache__.]

| Path | Purpose |
|------|---------|
| [dir/file] | [what it does] |
```

---

## Step 5 — Output

After writing:

```
CONTEXT.md [created|updated]
Sections: Architecture · Key Decisions · Known Limitations · Active Work · Do Not Touch · File Map
Next: Load this file at session start by adding to CLAUDE.md:
  "Read CONTEXT.md at the start of every session."
```

---

## Status

End every run with one of:

- **DONE** — CONTEXT.md written or updated
- **DONE_WITH_CONCERNS** — some sections are incomplete (state which ones and why)
- **BLOCKED** — cannot determine project context (no source files, no git history, no manifests)
- **NEEDS_CONTEXT** — project directory not specified
